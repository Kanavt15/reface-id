1. PROJECT OVERVIEW
================================================================================

Title:    REface ID — 3D Forensic Facial Reconstruction
Version:  1.0.0
Author:   REface ID Team
License:  ISC — For authorized forensic and law enforcement use

Description:
  A desktop application for 3D facial reconstruction used in forensic and law
  enforcement investigations. Upgrades the traditional 2D composite sketch
  workflow into an interactive 3D system powered by Blender Engine, Three.js,
  and AI-assisted face generation (Claude/Gemini).

Primary Use Cases:
  - Law enforcement facial composite creation from witness descriptions
  - Forensic identification and reconstruction from skeletal remains
  - Courtroom presentation of suspect/victim facial reconstructions
  - AI-assisted rapid face generation from natural language descriptions
  - 3D model export for printing, sharing, or further analysis

3.2 Process Lifecycle:
  1. Electron starts → spawns Flask backend (server.py on port 5001)
  2. BrowserWindow loads index.html (frameless, custom title bar)
  3. app.js initializes all systems: SceneManager, OBJMorpher, HairSystem,
     EyeSystem, BackendAPI, CaseManager, etc.
  4. Loads head.glb + head_regions.json concurrently
  5. OBJMorpher binds mesh; Hair/Eye systems attach to head
  6. UIController wires all DOM events, sliders, panels
  7. AI/Snapshot/HeadTracker systems initialized
  8. User interacts via sliders, AI chat, point editing, etc.
  9. Changes sync through CaseManager (undo/redo stack)
  10. Export/Save routes through Flask → Blender background scripts

3.3 IPC Bridge (preload.js):
  Exposed via window.electronAPI:
  - Window controls: minimize, maximize, close
  - Dialogs: saveDialog, openDialog, saveFile, downloadExportedFile
  - Menu callbacks: onNewCase, onOpenCase, onSaveCase, onExport, onScreenshot

5.1 FACE RECONSTRUCTION & MORPHING
  - Real-time landmark-based vertex deformation via OBJMorpher.js
  - 50+ morph parameters across 9 facial regions:
    * Skull: faceWidth, faceLength, headWidth, headLength, faceTaper
    * Forehead: foreheadHeight, foreheadSlope, foreheadWidth, templeWidth,
                foreheadBulge
    * Brows: browHeight, browSpacing, browProminence, browArch, browThickness
    * Eyes: eyeSpacing, eyeHeight, eyeDepth, eyeSize, eyeTilt, eyeOpenness
    * Nose: noseLength, noseWidth, noseBridgeWidth, noseBridgeHeight,
            noseTipHeight, noseTipWidth, nostrilFlare
    * Cheeks: cheekFullness, cheekboneProminence, cheekHeight, nasolabialDepth

    * Jaw/Chin: jawWidth, chinHeight, chinWidth, chinProtrusion, jawDefinition

8. MORPH ENGINE TECHNICAL DETAILS
================================================================================

8.1 OBJMorpher Architecture:
  - Loads GLB head mesh → extracts all sub-meshes
  - Stores original vertex positions (Float64Array)
  - 100+ facial landmarks with 3D coordinates (Y-up system)
  - Per-parameter morph rules define:
    * Target landmark(s)
    * Influence radius (Gaussian falloff)
    * Direction (X/Y/Z axis, scaling vs translation)
    * Side-awareness (left/right/both)
  - Face mask: smooth boundary at face edges (transition_width=0.075)
  - Weight caching for performance

8.2 Deformation Pipeline:
  1. User changes slider → morphValues[param] updated
  2. For each param, compute weight per vertex:
     weight = gaussian(distance_to_landmark) * face_mask * directional_weight
  3. Accumulate vertex offsets across all params
  4. Apply: final_pos = original_pos + offset
  5. Update Three.js geometry buffers
  6. Trigger onMorphApplied → refresh hair/eyes/marks

8.3 Web Worker Offloading:
  - MorphWorker.js runs heavy computation off main thread
  - Messages: init (send geometry data), morph (send values, receive positions)
  - Transferable Float32Arrays for zero-copy performance


9. AI SYSTEM PROMPT & PARAMETER MAPPING
================================================================================

The AI system prompt (in server.py) instructs Claude/Gemini to:
  - Accept natural language face descriptions
  - Return strict JSON with parameter deltas
  - Support all morph targets, hair styles, eyebrows, beard, appearance
  - Handle facial marks (scars, birthmarks, moles, pimples, wounds)
  - Support conversational refinement with relative adjustments
  - Analyze reference images for visible facial traits
  - Value guide: 0=minimum, 50=neutral, 100=maximum
  - "slightly" = 5-10 delta, "more" = 15-25 delta

Output format:
  {
    "morphTargets": { "paramName": value },
    "hair": { "style": "hair1", "color": "#hex", ... },
    "eyebrows": { "thickness": 50, ... },
    "beard": { "style": "none", "color": "#hex" },
    "appearance": { "skinColor": "#hex", "eyeColor": "#hex", ... },
    "facialMarks": [ { "type": "scar", "region": "cheek", ... } ]
  }

12. DEVELOPMENT WORKFLOW
================================================================================

12.1 Setup:
  Prerequisites: Node.js 18+, Python 3.9+, Blender 3.6+
  npm install                  # Install Node dependencies
  pip install -r backend/requirements.txt  # Install Python dependencies
  Copy .env.example to .env and add API keys

12.2 Running:
  npm run dev                  # Start backend + Electron together
  npm start                    # Electron only (backend must be running)
  npm run start:backend        # Python backend only

12.3 Building:
  npm run build                # electron-builder → dist/

12.4 Key Entry Points for Developers:
  - App behavior:     src/renderer/js/app.js, UIController.js
  - Morph quality:    src/renderer/js/OBJMorpher.js
  - Hair/eye:         src/renderer/js/HairSystem.js, EyeSystem.js
  - AI prompts:       backend/server.py (AI_SYSTEM_PROMPT), AIController.js
  - Blender pipeline: backend/blender_scripts/*.py
  - UI layout:        src/renderer/index.html
  - Styling:          src/renderer/styles/*.css

12.5 Adding New Models:
  - Hair: Export as HairN.glb → assets/models/hair/ → update HairSystem.js
  - Beard: Export as BeardN.glb → assets/models/facial/ → update HairSystem.js
  - Eyes: Export as GLB → assets/models/facial/ → update EyeSystem.js


13. SUMMARY
================================================================================

REface ID is a full-featured forensic 3D facial reconstruction desktop
application with:

  - 50+ parametric morph controls with real-time deformation
  - GLB-based hair (12 styles), eyebrow, beard, eye, and eyelash systems
  - Direct point-based sculpting for advanced users
  - Skin mark system (moles, scars, wounds) with surface tracking
  - AI-assisted face generation via Claude/Gemini with image analysis
  - Voice input for hands-free operation
  - Webcam head tracking for natural interaction
  - Full case management with undo/redo and state snapshots
  - Multi-format 3D export (OBJ/FBX/GLB) via Blender
  - Blender-powered high-quality rendering (Cycles/EEVEE)
  - Dark forensic-themed professional UI

The codebase is modular (~8,300 lines of JS, ~1,200 lines of Python) with
clear separation between rendering, logic, and backend systems. The architecture
supports incremental enhancement through well-defined system boundaries.
